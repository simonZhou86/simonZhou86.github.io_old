Common Machine Learning Engineer Interview Questions

Posted on 2020-07-03 Edited on 2020-07-04 In Interview Question Views: Valine:

Introduction:

Hello guys, Welcome back! In this article, I am going to share some common interview questions for the position of Machine Learning Engineer and Data Scientist. These questions are either the one appeared on my previous interviews or from the internet. The questions will mainly focus on Machine Learning Algorithms and Mathematical Derivations, but I will also include some Data Structures & Algorithms. Lots of these questions are for Internship and some are for New Grad position.

Machine Learning Algorithm Questions:

1. What is Supervised and Unsupervised Learning?

Supervised Learning are used for Classification or Regression, where there are training labels in dataset. Bias-Variance trade-off and Model complexity will be considered in Supervised Learning. Unsupervised Learning, however, does not require training labels, people usually wish to learn he inherent structure from the data.
Supervised Learning Algorithm: Linear Regression, Tree-Based Method, SVM…
Unsupervised Learning Algorithm: K-Means Clustering, Dimension Reduction

2. Bagging Vs. Boosting

Bagging: Idea of sample with replace. N new training data sets are produced by random sampling with replacement. Prediction rule of bagging is average the responses among all learners. It works well for high variance and low bias procedures. Support multi-processing for weak learners.
Boosting: Will use the same dataset for all epoch. Sequentially select prediction rule(New model is related to previous one). Boosting uses weighted average among all learners. It works well for low variance and high bias.

3. Random Forest Vs. GBDT:

Random Forest: can be formed by either regression tree or classification tree. Trees in Random Forest can be formed parallelly. Results from Random Forest are mostly generated by majority voting. Random Forest is not very sensitive with outliers and it treated each learner equally. Random Forest is improved by reducing variance.

GBDT: can be formed by only regression tree. Trees in GBDT can be formed sequentially. Results from GBDT is generated by weighted sum. GBDT is very sensitive with outliers and ensembled by weighted weak learners. GBDT is improved by reducing bias.

4. Why Bagging can reduce variance?

Bagging is a collection of some learners. Because bagging is the idea of sample with replace, these independent learners may result in overfitting (high variance). Therefore by averaging these learners can reduce variance.

5. Please derive GBDT algorithm and explain in what situation GBDT can be used:

loss function:
$$
F^* = argmin E_{x,y} [L(y,F(x))]
$$
$$
F(x;\rho_m,a_m) = \sum_{m=o}^{M} \rho_m h(x;a_m)
$$
for m=1 to N, do following,
$$
g_m(x) = - \frac{\partial E_y[L(y, F(x)) | x]}{\partial F(x)} \ evaluate \ at \ F(x) = F_{m-1}(x) \quad (1)
$$
$$
\rho_m = argmin E_y[L(y, F_{m-1}(x) + \rho g_m(x))|x] \quad (2)
$$
$$
f_m(x) = \rho_m g_m(x) \quad (3)
$$
$$
F_m(x) = F_{m-1}(x) + \rho_m g_m(x) \quad (4)
$$
End for loop. And then, we sum up all trees to get:
$$
F^*(x) \approx F_M(x) = F_0(x) + \sum_{m=1}^{M}\rho_m g_m(x)
$$

GBDT can used for regression problems, no matter it is linear or nonlinear.

6. How could you do if L1 is not differentiable?

When loss function is not differentiable, the gradient descent is then not applicable. In this case, we may use Coordinate descent algorithm. Coordinate descent algorithm is done by follow the direction of coordinate. If we have $p$ features, we may fix value of the first $p-1$ features, and find the optimal one for $p^{th}$ feature. Repeat the step until convergence.

7. How could you dealing with unbalanced data?

Large discrepancy between positive and negative data.
Pick the right score matrix. For example: use F1 score, precision score , AUC.
Other techniques: oversampling/undersampling (not recommended), use KNN algorithm to produce similar data(SMOTE)

8. What are some common activation functions in Neural Networks? What’s the purpose of standardization?

Commonly used activation functions: Sigmoid(logistic), TanH, ReLu, Softmax (for output layer)
Modified ReLu: leaky ReLu, Exponential Lu, Parametric ReLu
Purpose of standardization: improve accuracy and speed of gradient descent to find the solution

9. What is the difference between L1 and L2 regularization?

When adding a sum of the absolute value of all coefficients to loss function, this is L1.
When adding a sum of the square of all coefficients to loss function, this is L2.
$$
L1:\ L(\beta) = ||Y-X\beta||^2 + \lambda \sum_{j=1}^{p}|\beta_j|
$$
$$
L2: \ L(\beta) = ||Y-X\beta||^2 + \lambda \sum_{j=1}^{p}\beta_j^2
$$
L1 shrinks the less significant features’ coefficient to zero(sparsity), which has the same effect of variable selection.
L2 does not perform the same as L1, even if some coefficient is really small, L2 will keep it as the same. Mostly use for prevent overfitting.

10. Could L2 regularization solve for collinearity problem? Explain it mathematically

Consider a simple linear regression case: $Y = \theta_0 + \theta_1x_1 + … + \theta_nx_n + \epsilon$ Solve for $\hat{\theta}$ we have: $\hat{\theta}$= $(X^TX)^{-1}X^Ty$.
Collinearity may appears from the following scenario: if a column can be written as a linear combination other columns, then we may have a problem.
Mathematically, the matrix $X^TX$ is not full rank anymore, then it could not be inverted. Further means that the $\hat{\theta}$ may have infinitely many solutions.
However, if we add a L2 regularization term then minimized:
$$
g(\theta) = min(||Y-X\theta||^2 + \lambda||\theta||^2)
$$
Compute the partial derivation with respect to $\theta$ and set to 0:
$$
\nabla g(\theta) = 2X^TX\theta - 2X^TY + 2I\theta\lambda = 0
$$
implies:
$$
\hat{\theta} = (X^TX+\lambda I)^{-1} X^TY \quad (1)
$$
Where in equation 1, the matrix is invertible now. Therefore, L2 can solve collinearity problems.

3. How do you dealing with collision in Hash Table?

Open addressing technique. Generate new index in order to avoid collision. There are three ways:

  1. Linear Probing(Linked List): store all values that in the same index to a linked list.
  2. Quadratic Probing: construct the new index by adding successive values of a random polynomial until an open slot is found
  3. Double hash: use a second hash function to the current index until an open slot is found.

4. Java OOP Abstract Class Vs. Interface:

A Java class can implemented multiple interfaces, but it can only inheritance one abstract class. Functions in interface are default to public type, but in abstract class, functions can be protected. We can not define attributes in interface, but in abstract class we could.

5. SQL question: Difference between count(), count(colname) and count(*)?

count(1) and count() will not ignore NULL values, count(colname) will ignore NULL.
If column name is primary key, then count(colname) is faster than other two. If you want to count through multiple column and there is no primary key for these columns, count(1) will have the execution priority.
